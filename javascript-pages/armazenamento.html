Armazenamento em JavaScript - localStorage

Assim como os cookies a forma de armazenamento local localStorage permite que sejam guardados dados em pares chave-valor do usuário, onde os dados são persistidos sem uma data de validade, ou seja, somente são apagados caso o usuário limpe os dados;cache do navegador. Um característica é a de que o localStorage reduz a quantidade de tráfego da rede, pois os dados não são enviados de volta ao servidor.

Acessando o localStorage

Para acessar o localStorage do seu navegador basta digitar o tecla de atalho ctrl + shift + i, procurar a aba armazenamento e em seguida a opção com Local Storage como na imagem abaixo. Em alguns navegadores pode estar dentro da aba aplicativo ao invés de armazenamento

Os principais métodos

métodos/propriedades Descrição
key(n) Retorna o nome da chave na n-ésima posição do armazenamento local
length Retorna o número de itens armazenados localmente
getItem(chave) Retorna o valor da chave especificada
setItem(chave, valor) Adiciona um novo par de chave-valor dentro do armazenamento local, ou atualiza caso a chave já exista.
removeItem(chave) remove a chave especificada
clear() Esvazia todas as chaves do armazenamento

setItem() - armazenar

O método localStorage.setItem("chave", "valor"); armazena localmente um item de dados que leva como parâmetros a chave e seu respectivo valor. Abra o seu local storage e veja os valores preenchidos. Abra o seu armazenamento local e veja os dados

<script>
localStorage.setItem("Nome", "Fulano de Tal");
localStorage.setItem("idade", "67");
localStorage.setItem("sexo", "M");
</script>

getItem() - pegar

O método localStorage.getItem("chave"); retorna o valor, armazenado localmente, da chave especificada

Mostrar Código

<div id="get"></div>
<script>
var nome = localStorage.getItem("Nome");
document.getElementById("get").innerHTML = nome;
</script>

length - quantidade

O atributolocalStorage.length; retorna a quantidade de dados armazenados localmente.

Mostrar Código

<div id="length"></div>
<script>
var quantidade = localStorage.length;
document.getElementById("length").innerHTML = quantidade;
</script>

key - verificando

O métodolocalStorage.key(n); retorna o nome da chave que está na posição especificada.

Mostrar Código

<div id="key"></div>
<script>
var nomeChave = localStorage.key(1);
document.getElementById("key").innerHTML = nomeChave;
</script>

removeItem() - remover

O método localStorage.removeItem("chave");remove o par chave-valor, armazenado localmente, pelo nome da chave especificada. Abra novamente o seu armazenamento local e confira a exclusão.

<script>
localStorage.removeItem("Nome");
</script>

clear() - remover tudo

O método localStorage.clear();remove todos os dados!

Mostrar Código

<button onclick="apagar()">Limpar Armazenamento Local </button>
<script>
function apagar() {
localStorage.clear() ;
}
</script>

sessionStorage

O objeto sessionStorage é idêntico ao objeto localStorage, com a única diferença é que os dados ficam armazenados apenas durante o uso, ou seja, os dados são excluídos assim que o navegador é interrompido.

setItem() - armazenar

O método localStorage.setItem("chave", "valor"); armazena localmente um item de dados que leva como parâmetros a chave e seu respectivo valor. Abra o seu local storage e veja os valores preenchidos. Abra o seu armazenamento local e veja os dados

<script>
sessionStorage.setItem("Nome", "Fulano de Tal");
sessionStorage.setItem("idade", "67");
sessionStorage.setItem("sexo", "M");
</script>

localStorage X Cookies

A principal diferença entre o cookie e o localStorage é que os cookies são principalmente para leitura no lado do servidor (mesmo que possam haver scripts no lado do cliente), enquanto o localStorage só pode ser lido pelo lado do cliente.. A segunda diferença está no tamanho do armazenamento, enquanto o localStorage oferece 5MB de armazenamento o cookie oferece apenas 4KB.
