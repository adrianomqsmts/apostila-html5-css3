Funções em JavaScript

Uma função é um trecho de comandos em bloco que faz uma determinada tarefa que pode ser realizada em diferentes pontos de um código e quantas vezes quisermos. As vantagens de se utilizar funções é a clareza do código, pois podemos separar o código em diversas partes facilitando a manutenção do mesmo e a possibilidade de reutilização mais simples (sem ter copiar e colar varias linhas de códigos).

A sintaxe básica de uma função é: function nome(parametro1, paramentro2,...) { código; }. Onde function é a palavra chave para criar uma função com o nome especificado seguido sempre de parenteses que podem ou não conter parâmetros. Os parâmetros são variáveis que irão receber um valor e serem usada dentro do bloco de código da função.


function soma(a, b) {
return a + b;
}

Agora você pode ter notado um pouco melhor a vantagem de se usar variáveis, pois assim, podemos passar como parâmetro para função quaisquer valores e ela realizará a soma, e então através da palavra reservada return irá retorno exatamente no lugar onde está a função o resultada da operação de soma entre as variáveis a e b.

Podemos chamar uma função usando o seu nome e entre parenteses os valores das variáveis

Mostrar Código

<div id="funcao1"></div>
<script>
function soma(a, b) {
return a + b;
}
document.getElementById("funcao1").innerHTML = soma(10,20);
</script>

Escopos

Escopos são blocos de códigos que podem ou não conter outros blocos de códigos dentro deles, gerando níveis diferentes, consequentemente escopos diferentes, onde os escopos irão determinar a acessibilidade das variáveis. Dentro do conceito de escopos temos:

Variáveis locais

Quando declaramos uma variável dentro de uma função, chamados ela de variável local (escopo de função), ou seja, a variável está acessível somente dentro da função. Como as variáveis local somente são acessadas dentro de uma função então podemos usar o mesmo nome de variável em outro lugar e elas serão independentes uma da outra, mesmo que tenham o mesmo nome.

Uma variável dentro de uma função é criada durante a execução da função e excluída ao final da execução de cada função. O mesmo conceito de aplica a qualquer bloco de código como estruturas condicionais e de repetição.

Variáveis Globais

O conceito de variável glocal refere-se a variáveis criadas em escopos de níveis anteriores, ou no caso, criadas anteriormente a criação de uma função. Essas variáveis podem ser acessadas por todos os blocos de níveis inferiores à ela que estejam dentro do seu bloco local de declaração.

Veja no exemplo abaixo que temos a variável d que pode ser acessada pela função, a variável local e global c, mas somente o valor dentro da função será usado, enquanto o valor fora da função será conservado.

Mostrar Código

<div id="funcao2"></div>
<script>
var d = 10;
var c = 5;
function soma(a, b) {
var c = 10;
return a + b + c + d;
}
document.getElementById("funcao2").innerHTML = "Resultado da soma: " + soma(10,20 ) + "<br> valor da variável C: " + c;
</script>



Variáveis Let

As variáveis let possuem um comportamento de escopo mais interessante. Um novo escopo é delimitado pode {} mas existem casos em que o conceito de escopo não é definido, mas queremos que a variável se comporte como se estivesse dentro de um escopo.

Mostrar Código

<div id="escopo"></div>
<script>
var c = 5;
{
var c = 10;
}
document.getElementById("escopo").innerHTML = "<br> valor da variável C: " + c;
</script>



Para conservar o escopo precisamos criar uma variável let

Mostrar Código

<div id="let"></div>
<script>
var c = 5;
{
let c = 10;
}
document.getElementById("let").innerHTML = "<br> valor da variável C: " + c;
</script>



Dentro de uma função o comportamento entre var e let como você pode observar são semelhantes. Entretanto em estruturas de repetição e condicionais ou simples blocos de códigos, precismos usar a palavra reservada let para conservar o escopo.

Arrow function : Função seta

Arrow function é uma forma mais recente de se escrever funções de forma mais curtas, onde podemos omitir diversos detalhes da sintaxe de uma função regular.

function soma(a,b){
return a + b;
}

É o mesmo que:

soma => (a,b){
return a + b;
}

Em funções setas, se houverem apenas uma declaração de retorno então poderemos omitir a palavra reservada return e os parênteses.

soma = (a,b) => a + b;

Em funções setas que não possuem nenhum parâmetro:

texto = () => "Olá mundo!";

Em funções setas que possuem um único parâmetro, e somente neste caso, podemos ou não omitir os parênteses dos parâmetros:

texto = valor => "Olá mundo! " + valor;

texto = valor => "Olá mundo! " + valor;

Mostrar Código

<div id="arrow"></div>
<script>
var valor = "Boooom dia! Ou será boa tarde?! Ou Boa noooooite!? OH MDS!";
texto = valor => "Olá mundo! " + valor;
document.getElementById("arrow").innerHTML = texto(valor);
</script>
