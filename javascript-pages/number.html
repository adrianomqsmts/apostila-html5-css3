Números com JavaScript

Os números podem ser representados de algumas formas como decimal ou inteiro, como também pode ser escrito na notação científica.

<script>
var inteiro = 10;
var decimal = 0.10;
var cientifico = 10e2; // É o mesmo que 10 elevado a 2
var cientifico = 10e-2; // É o mesmo que 10 elevado a -2
</script>

Em JS todos os números são armazenados como pontos flutuantes de 64 bits, sendo assim, o maior número sen notação científica pode ser representado na linguagem é 10000000000000000. Enquanto o número máximo de casas decimais são 17 (0.00000000000000001) O que podem gerar pequeno erros que podem propagar em contas com valores muito pequenos e deve sempre tomar o devido cuidado (Para entender mais sobre esses erros deve-se pesquisar sobre números binários e erros).

Mostrar Código

<div id="erros"></div>
<script>
var soma = 0.1 + 0.2;
document.getElementById("erros").innerHTML = "A soma entre o número 0.1 e 0.2 é " +soma;
</script>


Existem algumas formas de tratar esses tipos de erros, a mais simples é transformar os valores decimais em inteiros e então reverter com noções básicas de matemática.

Mostrar Código

<div id="erro2"></div>
<script>
var soma = (0.1*10 + 0.2*10)/10;
document.getElementById("erro2").innerHTML = "A soma entre o número 0.1 e 0.2 é " +soma;
</script>


String x Numbers

Ao somarmos uma string e um número, o número será considerado uma string e então haverá uma concatenação.

Mostrar Código

<div id="erro3"></div>
<script>
var a = "10" ;
var b = 10 ;
var soma = a + b ;
document.getElementById("erro3").innerHTML = "Resultado: " +soma;
</script>


Outro erro comum é tentar somar junto com uma string que descreve a soma:

Mostrar Código

<div id="erro4"></div>
<script>
var a = 10 ;
var b = 10 ;
document.getElementById("erro4").innerHTML = "Resultado: " + a + b;
</script>


Entretanto, ao fazermos uma operação que não seja de soma (que pode ser confundida com concatenação) com duas string com números o JS tentará converter a string para números

Mostrar Código

<div id="multString"></div>
<script>
var a = "10" ;
var b = "10" ;
var mult = a * b ;
document.getElementById("multString").innerHTML = "Resultado: " + mult;
</script>


NaN - Não é um número

A palavra reservada NaN é usada para descrever um valor que não é um número válido. Geralmente ocorre quando tentamos usar uma operação (diferente de soma) entre uma string não numérica e um número

Mostrar Código

<div id="mult2"></div>
<script>
var a = "Obrigado" ;
var b = 10 ;
var mult = a * b ;
document.getElementById("mult2").innerHTML = "Resultado: " + mult;
</script>


Infinity : Infinito

O valor Infinity é retornado pelo KS sempre que uma operação ultrapassar o valor máximo que a Linguagem suportam ou então quanto ocorre alguma divisão por zero.

Mostrar Código

<div id="infinito"></div>
<script>
var a = 10/0;
document.getElementById("infinito").innerHTML = "Resultado: " + a;
</script>


Mostrar Código

<div id="infinito2"></div>
<script>
var a = 10 * 10e154;
document.getElementById("infinito2").innerHTML = "Resultado: " + a;
</script>


toString() : Números Hexadecimais e binários

Podemos escrever o número hexadecimal com a notação 0x na frente do valor hexadecimal. Outra forma é usar o método numero.toString(base), onde a base é da base numérica em que o número será convertido, podendo ser entre 2 a 32, onde a base 2 (binário), base 10 (decimal) e 16 (hexadecimal).

Mostrar Código

<div id="binario"></div>
<script>
var a = 9;
var binario = a.toString(2);
document.getElementById("binario").innerHTML = "Binário de 9 é: " + binario;
</script>


toExponential(): Notação Científica

O método numero.toExponential(casasDecimais) coloca o número na notação cientifica, arredondando o valor para o número de casas decimais especificado.

Mostrar Código

<div id="casasDecimais"></div>
<script>
var a = 9.156;
var casasDecimais = a.toExponential(2);
document.getElementById("casasDecimais").innerHTML = "Notação cientifíca de 9.156 com duas casas decimais é : " + casasDecimais;
</script>


Mostrar Código

<div id="casasDecimais2"></div>
<script>
var a = 9.156;
var casasDecimais2 = a.toExponential(5);
document.getElementById("casasDecimais2").innerHTML = "Notação cientifíca de 9.156 com 5 casas decimais é : " + casasDecimais2;
</script>


toFixed() : Arredondamento

O método numero.toFixed(casaDecimais) retorna uma string com o valor com o número de casas decimais arredondado a última casa decimal. toFixed(0) arredonda o número inteiro.

Mostrar Código

<div id="toFixed"></div>
<script>
var a = 9.156;
var arredondamento = a.toFixed(2);
document.getElementById("toFixed").innerHTML = "O número 9.156 com duas casas decimais é : " + arredondamento;
</script>


Mostrar Código

<div id="toFixed2"></div>
<script>
var a = 9.156;
var arredondamento = a.toFixed(0);
document.getElementById("toFixed2").innerHTML = "O número 9.156 arredondado: " + arredondamento;
</script>


toPrecision() : Cortando números

O método numero.toPrecision(quantidadeNumeros) faz a mesma coisa que o método toFixed(). Entretanto ele considera o tamanho total, incluindo os números inteiros e não somente as casas decimais

Mostrar Código

<div id="toPrecision"></div>
<script>
var a = 90.156;
var arredondamento = a.toPrecision(3);
document.getElementById("toPrecision").innerHTML = "O número 90.156 com tamanho 3 é: " + arredondamento;
</script>
